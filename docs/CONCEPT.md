# Deep Research Agent — Concept Document

## Vision

Персональный агент глубокого исследования для одного пользователя. Не enterprise-инструмент, а "цифровой исследовательский ассистент", который помнит контекст, умеет останавливаться для уточнений и структурирует результаты.

## Why This Project Matters

### Проблема существующих решений
- **Сложность setup**: Существующие open-source агенты (open_deep_research, deep-research) требуют 3-5 API ключей, Docker, сложной конфигурации
- **Отсутствие памяти**: Каждое исследование — изолированная сессия, нет истории
- **Нет human-in-the-loop**: Агент работает "вслепую", нельзя скорректировать курс
- **Enterprise-фокус**: Решения ориентированы на компании, а не на индивидуальных исследователей

### Наше отличие
> **Итеративное исследование с памятью**, как Notion/NotebookLM, но с активным агентом

| Фича | Существующие | Наш проект |
|------|--------------|------------|
| Human-in-the-loop | ❌ Нет | ✅ Обязательная остановка |
| Память между сессиями | ❌ Нет | ✅ SQLite история |
| Простота setup | ❌ 5+ env vars | ✅ Один config.toml |
| Источники | Только web | ✅ Web + GitHub + arXiv |
| Вывод | Markdown | ✅ JSON + Markdown |

## Core Principles

1. **Простота прежде всего** — MVP должен работать за 5 минут после клонирования
2. **Human-in-the-loop** — пользователь контролирует процесс на каждом шаге
3. **Персистентность** — исследования живут дольше сессии
4. **Модульность** — легко добавлять новые источники данных
5. **CLI-first** — пока не нужен веб, терминал быстрее

## Target Use Cases

1. **Быстрый обзор литературы** — для магистерской/статьи
2. **Изучение новой технологии** — собрать best practices из GitHub + документации
3. **Сравнительный анализ** — "сравни подходы X и Y"
4. **Поиск датасетов** — для ML-экспериментов

## Success Criteria

- [ ] MVP работает с Tavily (уже настроен)
- [ ] SQLite хранит историю исследований
- [ ] Можно продолжить исследование из прошлой сессии
- [ ] Human-in-the-loop между шагами
- [ ] Структурированный JSON-вывод + Markdown отчет

## Tech Stack

- **LangGraph** — stateful оркестрация
- **Tavily** — web search (уже интегрирован)
- **SQLite** — персистентность
- **Pydantic** — валидация структур
- **Rich** — красивый CLI вывод
- **Python 3.11+**

## Non-Goals (что НЕ делаем)

- Web UI (пока)
- Поддержка множества пользователей
- Real-time коллаборация
- Автоматическая публикация отчетов
